## ÐY"O Propósito da SPEC
A SPEC descreve **o que** o sistema deve fazer - sem definir *como* implementar.

Ela é o ponto central do **Spec-Driven Development (SDD)** e serve como:

- base para arquitetura  
- contrato para backend e frontend  
- referência para testes e QA  
- fonte para agentes de IA gerarem código automaticamente  

Toda mudança no comportamento do sistema **deve ser refletida nesta SPEC**.
