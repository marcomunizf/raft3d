# 02.04 - Requisitos Não Funcionais (RNF)

Os RNFs definem características de qualidade e restrições técnicas do sistema.

---

## RNF-01 - Usabilidade do painel admin

**Descrição**  
O painel administrativo deve operar com ações por modais e sem navegação complexa.

**Justificativa**  
O MVP exige fluxo rápido, com dashboard sempre acessível.

**Critérios de Aceite**  
- Ações principais (vendas, clientes, estoque) abrem em modais.  
- O dashboard permanece acessível durante o uso.

**Restrições / Considerações Técnicas**  
- Padrão de navegação por modais para todas as ações do MVP.

**Dependências**  
- RF-04, RF-05, RF-08, RF-09.

---

## RNF-02 - Controle de acesso

**Descrição**  
A plataforma administrativa deve ser restrita a administradores autenticados.

**Justificativa**  
Os dados de vendas, clientes e estoque são internos.

**Critérios de Aceite**  
- Sem login, não há acesso ao dashboard.  
- Login inválido exibe erro simples.

**Restrições / Considerações Técnicas**  
- Usuário padrão do MVP é hardcoded (admin/admin).

**Dependências**  
- RF-03.

---

## RNF-03 - Persistência relacional

**Descrição**  
Os dados do sistema devem seguir o schema relacional definido no projeto.

**Justificativa**  
O MVP depende de consistência entre vendas, pagamentos e estoque.

**Critérios de Aceite**  
- Tabelas e campos seguem o schema definido em `06-DatabaseSchema.md`.
- Todas as tabelas usam UUID como chave primária.

**Restrições / Considerações Técnicas**  
- Banco relacional PostgreSQL, conforme descrição do MVP.

**Dependências**  
- Documento `06-DatabaseSchema.md`.

---

## RNF-04 - Consistência de status e tipos

**Descrição**  
Valores de status e tipos devem seguir os conjuntos definidos no schema.

**Justificativa**  
Evita inconsistências em vendas, pagamentos e estoque.

**Critérios de Aceite**  
- Status de venda e pagamento seguem os valores do schema.
- Tipos de movimentação e categorias seguem os valores do schema.

**Restrições / Considerações Técnicas**  
- Conjuntos de valores definidos no schema do MVP.

**Dependências**  
- RF-06, RF-07, RF-09, RF-10.

---

## RNF-05 - Autenticacao e protecao de rotas

**Descricao**  
As rotas administrativas devem ser protegidas por autenticacao via JWT.

**Justificativa**  
O painel administrativo expoe dados internos e precisa de controle de acesso.

**Criterios de Aceite**  
- Login retorna um token JWT valido.
- Rotas protegidas recusam requisicoes sem token.

**Restricoes / Consideracoes Tecnicas**  
- JWT conforme definido em `04-TechStack.md`.

**Dependencias**  
- RF-03.
